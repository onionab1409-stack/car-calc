## 📌 ЖЕЛЕЗНЫЕ ПРАВИЛА РАБОТЫ

1. **Максимум возможностей** в каждой роли — никаких упрощений
2. **Планы с подпланами** — каждый этап разбит на подэтапы
3. **СТОП после каждого подэтапа** → жду команду «продолжай»
4. **Артефакты** — всё фиксируется в файлах, пушится в GitHub
5. **Контекст не теряется** — при переходе в новый чат создаётся файл переноса
6. **Строго последовательно** — P1 → P2 → P3 → ... никакой параллельности
7. **Пользователь не разработчик** — каждый шаг объясняется простым языком

### Подэтап 1.3 · API контракт
**Что делаю:**
- `docs/api-contract.md` — документация всех endpoints

**Endpoints:**
```
POST /api/calculate
  Body: { country, destination, price, currency, year, engineType, horsePower }
  Response: { totalRUB: number }
  Логика: валидация → расчёт → лог breakdown → вернуть только число

GET /api/rates
  Response: { USDT_RUB, KRW_RUB, CNY_RUB, updatedAt }
  Логика: кэшированные курсы

POST /api/lead
  Body: { telegramUserId, calcResult, phone?, comment? }
  Response: { success: true, leadId }
  Логика: сохранить заявку → уведомить в Telegram-группу
```

**Файлы на выходе:** `docs/api-contract.md`
